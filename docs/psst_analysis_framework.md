## 2. PSST 섹션별 체크리스트

### P (Problem) - 문제 인식 [배점: 20점]

#### 필수 요소
- [ ] 타겟 고객 명확히 정의
- [ ] 고객이 겪는 "심각한" 문제 서술
- [ ] 기존 솔루션의 한계점 명시
- [ ] 정량적 데이터 (통계, 설문, 인터뷰)

#### 성공 패턴
```
✓ "100명 인터뷰 결과 78%가 불편함 호소"
✓ "연간 XX억원 손실 발생하는 문제"
✓ "기존 솔루션 A는 OO 한계, B는 XX 한계"
```

#### 실패 패턴
```
✗ "많은 사람들이 불편해함" (막연함)
✗ "시장이 크다" (구체성 부족)
✗ 창업자 관점의 문제 정의
```

---

### S (Solution) - 해결 방안 [배점: 30점]

#### 필수 요소
- [ ] 핵심 제공 가치 명확화
- [ ] 기술/서비스 차별점
- [ ] MVP/프로토타입 증거
- [ ] 개발 타임라인 (간트차트)
- [ ] 지식재산권 현황/계획

#### 성공 패턴
```
✓ P에서 문제 3개 → S에서 해결책 3개 (1:1 매칭)
✓ "베타 테스트 결과 만족도 4.2/5.0"
✓ "처리 속도 기존 대비 3배 향상"
✓ 스크린샷/데모 영상 포함
```

#### 실패 패턴
```
✗ 기술 나열만 (왜 좋은지 설명 없음)
✗ "혁신적인", "획기적인" 형용사 남발
✗ 실제 구현 증거 없이 아이디어만
```

---

### S (Scale-up) - 성장 전략 [배점: 30점]

#### 필수 요소
- [ ] 비즈니스 모델 (수익 구조)
- [ ] 시장 규모 (TAM/SAM/SOM)
- [ ] 시장 진입 전략 (3단계)
- [ ] 구체적 영업/마케팅 채널
- [ ] 자금 운용 계획

#### 성공 패턴
```
✓ "1단계: 대학생 타겟 → 2단계: 직장인 확대 → 3단계: 기업 B2B"
✓ "월 구독료 XX원 × 목표 고객 YY명 = 연 매출 ZZ원"
✓ 채널별 CAC/LTV 계산
```

#### 실패 패턴
```
✗ "바이럴 마케팅으로 성장" (구체성 없음)
✗ 자금 소요만 나열 (시장진입 전략 없음)
✗ 비현실적 시장 점유율 목표
```

---

### T (Team) - 팀 구성 [배점: 20점]

#### 필수 요소
- [ ] 대표자 전문성/경력
- [ ] 팀원 역할 분담 (개발/영업/운영)
- [ ] 팀의 차별적 역량
- [ ] 추가 인력 채용 계획
- [ ] 외부 자문/네트워크

#### 성공 패턴
```
✓ "대표: 해당 산업 5년 경력, 관련 석사"
✓ "개발팀장: A사 시니어 개발자 출신"
✓ "자문위원: 업계 전문가 3인"
✓ "향후 6개월 내 마케팅 담당자 2명 채용 예정"
```

#### 실패 패턴
```
✗ 단순 이력 나열 (왜 이 팀인지 설명 없음)
✗ 1인 창업인데 역량 부족 분야 언급 없음
✗ 팀원들이 퇴사 안 한 이유 설명 못함
```

## 3. 합격/불합격 키워드 분석

### 합격 키워드 (긍정 신호)
| 카테고리 | 키워드 |
|----------|--------|
| 검증 | MVP, 베타테스트, 파일럿, POC, 고객검증 |
| 수치 | N명 인터뷰, N% 성장, N배 효율, N원 절감 |
| 팀 | 경력 N년, 전문가, 출신, 수상 이력 |
| 차별화 | 특허, 독자기술, 최초, 유일 |

### 불합격 키워드 (위험 신호)
| 카테고리 | 키워드 |
|----------|--------|
| 막연함 | 매우, 획기적인, 혁신적인, 엄청난 |
| 계획만 | ~할 예정, ~할 계획, 추후 진행 |
| 과장 | 최초, 세계 최고, 시장 지배 |

## 4. 발표평가 대비 가이드

### 발표 구조 (5분)
```
1. 훅 (30초): 문제 상황 임팩트 있게
2. 문제-해결 (1분): P-S 핵심만
3. 시장-성장 (1분): 숫자 중심
4. 팀 (30초): 왜 우리 팀인가
5. 요청 (1분): 자금 용도, 기대 성과
```

### 자주 나오는 질문 (질의응답 10분)
| 질문 유형 | 예시 |
|----------|------|
| 시장 | "경쟁사와 차별점이 뭔가요?" |
| 팀 | "팀원들은 왜 퇴사 안 했나요?" |
| 실행 | "지금까지 한 건 뭔가요?" |
| 수익 | "언제 BEP 도달하나요?" |
| 리스크 | "가장 큰 리스크는 뭔가요?" |

## 5. AI 자동 생성 시 적용 규칙

### 입력 → 출력 변환 규칙

```python
# 형용사 → 수치 변환
"큰 시장" → "연 {X}억원 규모, 연평균 {Y}% 성장"
"빠른 처리" → "{X}ms 처리, 기존 대비 {Y}배 향상"
"많은 고객" → "{X}명 잠재고객, {Y}명 실사용자"

# P-S 매칭 강제
if len(problems) != len(solutions):
    warning("문제와 해결책 개수 불일치")

# 두괄식 구조 강제
for section in ["P", "S1", "S2", "T"]:
    section.summary = first_sentence  # 핵심 결론 먼저
    section.details = remaining       # 상세 설명은 뒤에
```

### 필수 데이터 입력 가이드
```
[사용자 입력 필수]
- 아이템 한 줄 설명
- 타겟 고객 (구체적으로)
- 해결하려는 문제 (3개 이내)
- 차별화 포인트
- 팀 구성 (역할별)

[AI가 요청할 정량 데이터]
- 시장 규모 출처
- 고객 검증 데이터 (인터뷰/설문)
- MVP 진행 상황
- 팀원 경력 상세
```

## 6. 업종별 템플릿 커스터마이징

### IT/플랫폼
- Problem: 디지털 전환, 비효율, 정보 비대칭
- Solution: 플랫폼, 자동화, AI 활용
- Scale-up: B2C → B2B, 구독 모델
- Team: 개발자 비중 50% 이상

### 제조/하드웨어
- Problem: 기존 제품 한계, 생산 비용
- Solution: 신기술, 특허, 시제품
- Scale-up: 양산 계획, 파트너십
- Team: 엔지니어, 생산 전문가

### 바이오/헬스케어
- Problem: 의료 접근성, 진단/치료 한계
- Solution: 신기술, 임상 데이터
- Scale-up: 규제 대응, 라이선싱
- Team: 의료 전문가, 연구진

### 식품/F&B
- Problem: 건강, 편의성, 지속가능성
- Solution: 신소재, 새로운 경험
- Scale-up: 유통 채널, 브랜딩
- Team: 식품 전문가, 마케터
