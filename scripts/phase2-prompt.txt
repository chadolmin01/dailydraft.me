### 1. API 에러 응답 통일 — ApiResponse 패턴 전환 (최우선, 가장 큰 작업)

현재 상태:
- 34개 라우트는 이미 `ApiResponse.*` 사용 (src/lib/api-utils.ts에 정의)
- 47개 라우트는 raw `NextResponse.json({ error: '...' })` 사용
- 클라이언트가 두 가지 에러 형식을 모두 처리해야 하는 혼란

수행할 작업:
- 먼저 src/lib/api-utils.ts를 읽고 ApiResponse 패턴 파악
- app/api/ 디렉토리의 모든 route.ts 파일을 순회
- raw NextResponse.json({ error: ... }) 패턴을 찾아서 ApiResponse로 교체:
  • `NextResponse.json({ error: '인증이 필요합니다' }, { status: 401 })` → `ApiResponse.unauthorized()`
  • `NextResponse.json({ error: '...' }, { status: 400 })` → `ApiResponse.badRequest('...')`
  • `NextResponse.json({ error: '...' }, { status: 404 })` → `ApiResponse.notFound('...')`
  • `NextResponse.json({ error: '...' }, { status: 403 })` → `ApiResponse.forbidden('...')`
  • `NextResponse.json({ error: '...' }, { status: 500 })` → `ApiResponse.internalError()`
  • `NextResponse.json({ error: error.message }, { status: 500 })` → `ApiResponse.internalError()` (error.message 절대 노출 금지)
- 각 파일 상단에 import 추가: `import { ApiResponse } from '@/src/lib/api-utils'`
- 이미 ApiResponse를 사용 중인 파일은 건드리지 마
- 성공 응답 `NextResponse.json(data)` 또는 `NextResponse.json(data, { status: 200/201 })`은 변경하지 마

작업 단위:
- 5~10개 파일씩 묶어서 커밋 (한 번에 전부 하지 말 것)
- 각 묶음마다 next build 실행
- 빌드 실패 시 해당 묶음만 되돌리고, 실패 파일 기록 후 다음 묶음으로

주의사항:
- catch 블록에서 `error.message`를 클라이언트에 반환하는 부분은 반드시 제거
- `console.error`로 서버 로그에만 남기고, 클라이언트에는 일반적인 메시지만 반환
- 비즈니스 로직은 절대 건드리지 마. return 문의 응답 형식만 변경

### 3. 온보딩 쿠키 서명 (1시간)

현재 상태:
- middleware.ts에서 `onboarding_completed` 쿠키를 plain text "true"로 설정
- 브라우저 DevTools에서 직접 쿠키를 "true"로 설정하면 온보딩 건너뛰기 가능

수행할 작업:
- src/lib/cookie-signature.ts 새로 생성
- HMAC-SHA256 기반 서명/검증 함수:
```typescript
import { createHmac } from 'crypto'

const SECRET = process.env.CRON_SECRET || 'fallback-secret-key'

export function signValue(value: string): string {
  const signature = createHmac('sha256', SECRET).update(value).digest('hex')
  return `${value}.${signature}`
}

export function verifySignedValue(signedValue: string): string | null {
  const lastDot = signedValue.lastIndexOf('.')
  if (lastDot === -1) return null
  const value = signedValue.slice(0, lastDot)
  const signature = signedValue.slice(lastDot + 1)
  const expected = createHmac('sha256', SECRET).update(value).digest('hex')
  if (signature !== expected) return null
  return value
}
```
- middleware.ts에서 쿠키 설정 시 signValue('true') 사용
- middleware.ts에서 쿠키 검증 시 verifySignedValue() 사용
- 온보딩 완료 처리하는 곳 (Onboarding.tsx 또는 관련 API)에서도 서명된 값으로 쿠키 설정

주의사항:
- CRON_SECRET 환경변수를 서명 키로 재사용 (새 환경변수 추가 불필요)
- crypto는 Node.js 내장 모듈이므로 패키지 설치 불필요
- middleware.ts는 Edge Runtime이므로 crypto.subtle 사용이 필요할 수 있음
  → Edge에서 createHmac 사용 불가능하면 Web Crypto API(crypto.subtle.sign)로 대체
  → 이 경우 async 함수로 변경 필요
