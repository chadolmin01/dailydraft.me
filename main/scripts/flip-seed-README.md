# FLIP 시드 데이터 요청 가이드

Draft에 FLIP 1기~현재 기수 멤버 데이터를 "ghost member"로 미리 심기 위한 CSV 템플릿입니다.

## 왜 필요한가

- Draft 샘플 클럽(`/club/flip`)에 **진짜 5년치 기수**가 쌓인 모습을 보여주기 위해
- 알럼나이 연결 기능을 데모용 가짜가 아닌 **실데이터**로 시연하기 위해
- 1기 멤버가 나중에 Draft 가입하면 "이 계정이 당신 같습니다" → 클레임 → 실명 전환 (바이럴 훅)

## 개인정보 원칙 (중요)

### 받는 필드 (6개만)
| 필드 | 설명 | 예시 |
|---|---|---|
| `name` | 실명 (DB에만 저장, 공개 UI는 익명화) | 홍길동 |
| `cohort` | 기수 | 1기, 2기, 3기 |
| `track` | 직렬/파트 | 개발, 디자인, 기획, 비즈니스 |
| `university` | 소속 학교 | 경희대학교 |
| `role` | 당시 역할 | owner, admin, member, alumni |
| `projects` | 참여 프로젝트명 (쉼표 구분) | 밋플,학식알리미 |
| `notes` | 운영진이 기억할 메모 (선택) | "창립 회장", "2기 부회장" |

### 절대 받지 않는 필드
- 이메일, 전화번호, 주소
- 프로필 사진
- 학번, 생년월일
- 기타 식별 가능한 민감정보

**이유**: 받는 순간 보안 책임과 법적 리스크가 커집니다. 위 6개만으로 Draft의 모든 데모/검증이 가능합니다.

## 공개 범위 정책

1. **기본값**: 이름은 `"멤버 A-03"` 같은 익명 ID로만 공개. 기수/직렬/프로젝트 기여는 표시.
2. **본인 클레임 시**: 해당 멤버가 Draft에 가입 + "이 ghost 계정이 저입니다" 클레임 → 실명 공개로 전환.
3. **이의 제기**: 본인이 삭제 요청 시 즉시 해당 행 삭제.

## CSV 작성 규칙

- 인코딩: UTF-8
- 구분자: 쉼표 (`,`)
- 여러 프로젝트는 `"밋플,학식알리미"` 처럼 전체를 큰따옴표로 감싸고 내부는 쉼표
- role 값은 반드시 `owner` / `admin` / `member` / `alumni` 중 하나
- cohort는 자유 형식 텍스트 — `1기`, `2021-1`, `2025 Spring` 뭐든 OK
- 모르는 값은 빈 칸으로 두기 (null로 처리됨)

## 파일 경로

작성한 CSV를 다음 경로에 저장:
```
main/scripts/flip-seed.csv
```

`flip-seed-template.csv`는 예시이고, 실데이터는 `flip-seed.csv`로 저장합니다. `.gitignore`에 추가되어 커밋되지 않습니다.

## 시드 실행

```bash
cd main
node scripts/seed-flip.mjs
```

(시드 스크립트는 별도 작업으로 추가 예정)
