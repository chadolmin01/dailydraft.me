# Draft — 최종 서비스 계획서

> 최종 수정: 2026-03-11
> 상태: MVP 리포커싱 (기능 축소 + 핵심 루프 집중)

---

## 1. 한 줄 정의

**"프로젝트 올리고, 사람 찾고, 커피챗으로 만나는 플랫폼"**

대학생/청년 창업자가 사이드 프로젝트를 공유하고, 함께할 팀원을 찾고, 커피챗으로 직접 만나는 곳.

---

## 2. 핵심 루프

```
프로젝트 올리기 → 사람 찾기 → 커피챗으로 만나기
     ↑                                    |
     └────── 주간 업데이트로 생동감 ────────┘
```

사용자가 5분 안에 가치를 느낄 수 있어야 한다.
- 비로그인: 프로젝트 둘러보기 → 관심 가는 프로젝트 발견 → 가입 유도
- 로그인: 프로젝트 올리기 or 커피챗 신청 → 실제 만남 발생

---

## 3. 타겟 사용자

| 구분 | 설명 |
|------|------|
| 1차 | 대학생 사이드 프로젝트 팀원을 찾는 사람 |
| 2차 | 초기 스타트업 co-founder를 찾는 사람 |
| 공통 | "아이디어는 있는데 같이 할 사람이 없는" 상태 |

## 4. 남기는 기능 (Core)

### 4-1. 프로젝트 공유
- 프로젝트 등록 (제목, 설명, 필요 역할, 기술 스택, 태그)
- 프로젝트 목록 탐색 (필터, 정렬)
- 프로젝트 카드에 "최근 업데이트 N일 전" 배지 — 살아있는 프로젝트 vs 죽은 프로젝트 즉시 구분
- 프로젝트 상세 모달 (Notion 스타일, macOS 윈도우 바)
- 주간 업데이트 (프로젝트 오너가 진행 상황 기록)
- 댓글/피드백 (프로젝트에 대한 커뮤니티 의견)

**DB 테이블:** `opportunities`, `project_updates`, `comments`, `helpful_votes`

### 4-2. 사람 찾기
- 프로필 등록 (닉네임, 대학, 포지션, 스킬, 관심 태그)
- 공개 프로필 탐색 (Explore 사람 탭)
- AI 프로필 요약 (한 줄 요약: "프론트엔드 중심 개발자, 헬스케어 관심")
  - 유지 이유: 사용자 수가 적어도 프로필 훑어볼 때 판단을 도움
  - 범위: 요약만. "매칭 확률 85%" 같은 AI 추천/매칭은 제거

**DB 테이블:** `profiles` (+ `profile_analysis` JSONB 필드)

### 4-3. 커피챗
- 프로젝트 상세에서 "커피챗 신청" → 메시지 템플릿 선택 or 직접 작성 → 오너에게 요청
  - 템플릿 예시: "이 프로젝트에 관심이 있어요, 이야기 나눠보고 싶습니다" / "OO 역할로 참여하고 싶어요" / 직접 작성
  - 빈 칸 앞에서 멈칫하는 허들을 낮춤
- 프로필에서 직접 커피챗 신청도 가능
- 오너가 수락하면 연락처 공개
- **기존 applications(지원) 시스템을 커피챗으로 통합**
  - "이 프로젝트에 관심 있어요 + 대화하고 싶어요" = 커피챗 하나로 처리
  - 프로젝트 오너의 커피챗 요청 목록 = 지원자 목록
- **알림 (핵심 루프의 일부)**
  - 커피챗 요청 시 → 오너에게 이메일 알림
  - 수락/거절 시 → 요청자에게 이메일 알림
  - 48시간 미응답 시 → 오너에게 리마인더
  - 알림이 안 가면 커피챗은 죽은 요청이 됨
  - **이메일 발송: Resend** (Supabase 기본 이메일은 스팸함 리스크). Edge Function으로 처리

**DB 테이블:** `coffee_chats`, `notifications`

### 4-4. 인증
- 이메일/비밀번호 가입
- Google OAuth, GitHub OAuth
- 온보딩 (프로필 기본 정보 입력)

**DB 테이블:** `auth.users`

## 5. 제거하는 기능 (UI에서 숨김, DB는 유지)

| 기능 | 이유 | 처리 방법 |
|------|------|-----------|
| 스타트업 아이디어 (PH/YC 분석) | 핵심 루프에 안 들어감. 콘텐츠는 인스타 재활용 | Explore에서 아이디어 탭/캐러셀/그리드 제거 |
| 사업계획서 AI 생성 | PMF 전에 불필요 | 네비게이션에서 제거 |
| 아이디어 밸리데이터 | 위와 동일 | 네비게이션에서 제거 |
| 구독/결제 (Toss) | 유료 전환 시기 아님 | Premium 뱃지, 초대코드 UI 숨김 |
| AI 매칭/추천 | 유저 100명 미만에서 의미 없음 | 프로필 사이드바 AI 매칭 분석 패널 제거 |
| applications 테이블 | 커피챗으로 대체 | UI에서 "지원하기" → "커피챗 신청"으로 통합 |
| 대시보드 페이지들 | 핵심 루프 밖 | 네비게이션에서 제거 |

### 제거 대상 페이지 (네비게이션에서 숨김)
- `/dashboard` — 불필요
- `/network` — 불필요
- `/messages` — 불필요 (커피챗이 대체)
- `/calendar` — 불필요
- `/documents` — 불필요
- `/business-plan` — 제거 대상
- `/validated-ideas` — 제거 대상
- `/workflow` — 불필요
- `/usage` — 제거 대상 (구독 관련)
- `/idea-validator` — 제거 대상

### 제거 원칙
- **코드 삭제 X, UI에서 숨기기만** → 롤백 가능
- **DB 테이블 DROP X** → 안 쓰면 비용 안 듦, 리스크만 큼
- 나중에 유저가 쌓이면 기능 복구 가능

## 6. 페이지 구조 (리포커싱 후)

```
/ (랜딩)
├── /login (로그인/가입)
├── /onboarding (프로필 설정)
├── /explore (메인 — 탭 2개)
│   ├── 프로젝트 탭 (기본)
│   └── 사람 탭
├── /profile (내 프로필 + 내 프로젝트 + 받은 커피챗)
├── /projects/new (프로젝트 등록)
└── /p/[id] (프로젝트 공유 URL — 풀페이지, 바이럴 루프 진입점)
```

### 네비게이션 (TopNavbar)
```
[Draft 로고]   탐색    마이페이지    [+ 새 프로젝트]   [알림]   [프로필]
```
- "탐색" = `/explore`
- "마이페이지" = `/profile`
- "프로젝트" 별도 탭 제거 → Explore에 통합

## 7. Explore 페이지 재설계

### 현재 (제거 전)
- 스타트업 아이디어 캐러셀 (상위 6개)
- 카테고리 사이드바 (AI/ML, SaaS, Fintech...)
- 프로젝트 섹션 (4개)
- 인재 섹션 (4개)
- 스타트업 아이디어 그리드 (12개)
- 트렌딩 태그, 필터, 정렬

### 변경 후
```
[프로젝트 탭] [사람 탭]
─────────────────────────
검색바 + 필터(카테고리, 역할, 기술스택)
─────────────────────────
프로젝트 탭:
  - 프로젝트 카드 그리드 (4열)
  - 정렬: 최신 / 인기 / 마감임박
  - 필터: 카테고리, 모집 역할, 기술스택

사람 탭:
  - 프로필 카드 그리드 (4열)
  - AI 요약 한 줄 표시
  - 필터: 포지션, 관심 분야, 대학
```

## 8. 프로필 페이지 재설계

### 현재 (제거 전)
- 프로필 정보 + 스킬 + 성격
- 내 프로젝트 목록
- 받은 커피챗
- AI 매칭 분석 사이드바 (파운더 타입, 추천 분야, 강점/성장점)
- 첨부파일

### 변경 후
- 프로필 정보 + AI 요약 한 줄 + 스킬
- 내 프로젝트 목록 + 주간 업데이트 현황
- 받은 커피챗 (= 지원자 관리)
- **AI 매칭 분석 사이드바 제거** (AI 요약은 프로필 본문에 표시)
- 첨부파일 유지

## 9. 사용자 플로우

### 신규 유저 (비로그인) — 랜딩 경유
```
랜딩 페이지 도착
  → "지금 올라온 프로젝트" 섹션에서 흥미로운 프로젝트 발견
  → 프로젝트 카드 클릭 → 상세 모달
  → "커피챗 신청" 클릭 → 가입 유도 CTA
  → 가입 → 온보딩 (프로필 작성)
  → 커피챗 신청 완료
```

### 신규 유저 (비로그인) — 공유 URL 경유 (바이럴 루프)
```
인스타 스토리/에타에서 /p/[id] 링크 클릭
  → 풀페이지 프로젝트 상세 (OG 프리뷰로 이미 관심 생긴 상태)
  → "커피챗 신청" 클릭 → 가입 유도 CTA
  → 가입 → 온보딩 → 커피챗 신청 완료
  → (선순환) 본인도 프로젝트 올림 → 자기 링크 공유
```

### 프로젝트 오너
```
가입/로그인
  → "새 프로젝트" 클릭
  → 프로젝트 등록 (제목, 설명, 필요 역할)
  → Explore에 노출
  → 커피챗 요청 수신 → 수락/거절
  → 매주 주간 업데이트 작성 → 프로젝트 생동감
```

### 팀원 지원자
```
가입/로그인
  → Explore에서 프로젝트 탐색
  → 관심 프로젝트 클릭 → 상세 확인
  → "커피챗 신청" + 메시지 작성
  → 오너가 수락하면 연락처 공개 → 실제 만남
```

### 사람 찾기
```
Explore → 사람 탭
  → AI 요약으로 빠르게 훑기
  → 관심 있는 프로필 클릭 → 상세
  → 커피챗 신청
```

## 12. 실행 계획

### Phase 1: UI 정리 + 핵심 플로우 확보
> 목표: 불필요한 것 숨기고, 비로그인 유저가 들어와서 가입까지 가는 길을 열어두기

1. TopNavbar에서 "프로젝트" 링크 제거, Explore에 통합
2. Premium/초대코드 UI 숨기기
3. Explore.tsx에서 스타트업 아이디어 관련 UI 전부 제거 (캐러셀, 그리드, 카테고리)
4. Explore.tsx를 프로젝트 탭 + 사람 탭 구조로 재설계
5. 프로필 페이지에서 AI 매칭 분석 사이드바 제거
6. "지원하기" 버튼 → "커피챗 신청"으로 통합
7. **비로그인 사용자 플로우** — 가입 없이 프로젝트 둘러보기 + 상세 모달 열기 가능하게. 커피챗/댓글 시도 시 가입 유도 CTA
8. **커피챗 알림** — Resend로 이메일 발송 (Edge Function). 요청/수락/거절 시 알림. 48시간 미응답 리마인더
9. **커피챗 메시지 템플릿** — 신청 시 템플릿 2~3개 드롭다운 제공 (허들 낮추기)
10. **프로젝트 카드 "최근 업데이트" 배지** — `project_updates` 최신 날짜 기준 "N일 전 업데이트" 표시

### Phase 1에 포함: 공유 URL `/p/[id]` 바이럴 루프
> 프로젝트 오너가 자기 링크를 인스타/에타에 공유 → 비로그인 유저 유입 → 가입

13. `/p/[id]` OG 메타 태그 — 프로젝트 제목, 설명, 필요 역할이 링크 프리뷰에 깔끔하게 표시
14. OG 이미지 동적 생성 — 프로젝트 카드 스타일의 이미지 (Next.js `opengraph-image` 또는 Vercel OG)
15. 풀페이지에서 바로 커피챗 신청 가능 (비로그인이면 가입 유도)
16. "이 프로젝트 공유하기" 버튼 — 링크 복사 + SNS 공유 (프로젝트 오너가 쉽게 뿌릴 수 있게)
17. 이게 되면 FLIP/CEREAL 회원이 자발적으로 홍보하는 구조가 만들어짐

### Phase 1.5: 콜드스타트 (초기 콘텐츠 시딩)
> 목표: 서비스 열었을 때 텅 빈 느낌 안 나게. 프로젝트가 최소 10~15개는 있어야 함

18. FLIP/CEREAL 동아리 실제 프로젝트 등록 유도 (직접 요청)
19. 본인 프로젝트 최소 2~3개 직접 등록 (Draft 자체 프로젝트 포함)
20. 온보딩 완료한 멤버에게 "프로젝트 올리기" 넛지 (첫 화면 CTA)
21. 랜딩 페이지에 실제 프로젝트가 보이도록 OpportunitySection 연결 확인

### Phase 2: 핵심 루프 완성

## 14. 성공 지표

### 핵심 지표 (이것만 봐도 됨)
| 지표 | 목표 | 왜 중요한가 |
|------|------|-------------|
| 커피챗 → 팀 결성 전환율 | 10%+ | 플랫폼이 실제로 팀을 만들어주는가의 증거. 이게 0이면 서비스가 죽은 거 |
| 48시간 내 응답률 | 70%+ | 응답이 안 오면 요청자가 이탈. 커피챗의 생명은 속도 |
| 커피챗 수락률 | 30%+ | 커피챗 플랫폼 평균이 20~30%. 50%는 비현실적, 30%면 건강한 수치 |

### 보조 지표
| 지표 | 목표 | 비고 |
|------|------|------|
| 핵심 루프 완주율 | 20%+ | 가입 → 커피챗 신청까지. 퍼널 어디서 빠지는지 추적 |
| 주간 업데이트 작성률 | 30%+ | 프로젝트 생동감. 업데이트 없는 프로젝트는 죽어 보임 |
| 재방문율 (주간) | 40%+ | 습관 형성 여부 |

| 지표 | 비고 |
|------|------|
| 등록 프로젝트 수 | 동아리 부탁이면 나오는 숫자. 단독으로 의미 없음 |
| 가입자 수 | 가입만 하고 안 쓰면 의미 없음 |
